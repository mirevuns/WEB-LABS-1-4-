## Файлы

- `server.py` - сервер
- `client.py` - консольный клиент

## Запуск сервера

python server.py --host 0.0.0.0 --port 9000

Файлы сохраняются в `server_storage`.

**Клиент** (локально):

```bash
python client.py --host 127.0.0.1 --port 9000
```

В сети: `--host` = IP компьютера с сервером.

## Команды

`LIST` - Список файлов на сервере 
`UPLOAD <путь>` - Загрузить локальный файл
`DOWNLOAD` - Скачать с сервера 
`EXIT` - Отключиться 
`HELP` - хелпа

## Протокол

Строки UTF-8, конец `\n`. Перед телом файла всегда передаётся размер в байтах.

- **LIST** - `OK`, число N, N имён
- **UPLOAD** - `READY` - размер - байты - `OK` (перезапись, пустой файл: размер 0)
- **DOWNLOAD** - `OK <размер>` + байты или `ERROR`
- **EXIT** - `BYE`

Сервер не падает при ошибках и обрыве соединения; несуществующий файл при скачивании - `ERROR файл не найден`.

## 1. Функции

### server.py

**safe_basename(name)** - из строки команды оставляет только имя файла. Отсекает `..`, пустые имена и попытки указать путь вне каталога хранения. Возвращает имя или `None`.

**recv_line(conn)** - читает из сокета одну текстовую строку до символа `\n`. Если клиент закрыл соединение - `ConnectionError`. Если строка длиннее лимита - `ValueError`.

**recv_exact(conn, n)** - читает ровно `n` байт (тело файла). `recv` может вернуть меньше байт за один вызов, поэтому чтение идёт в цикле, пока не наберётся `n`.

**send_line(conn, text)** - отправляет строку в UTF-8 с `\n` в конце.

**handle_list(conn, storage)** - обрабатывает LIST: собирает имена файлов из папки `storage`, шлёт `OK`, число файлов, затем каждое имя отдельной строкой.

**handle_upload(conn, storage, remote_name)** - обрабатывает UPLOAD: проверяет имя, отвечает `READY`, принимает строку с размером, затем байты файла, записывает в `storage` (режим `wb`, старый файл перезаписывается), отвечает `OK` или `ERROR`.

**handle_download(conn, storage, remote_name)** - обрабатывает DOWNLOAD: если файла нет - `ERROR`; иначе строка `OK <размер>`, затем содержимое файла блоками по `CHUNK`.

**handle_client(conn, addr, storage)** - работа с одним подключённым клиентом. В цикле читает команду, вызывает нужный обработчик. При `EXIT` шлёт `BYE` и выходит. Ошибки и обрыв закрывают только эту сессию, не весь сервер.

**ensure_storage(path)** - создаёт папку хранения, если её ещё нет.

**serve(host, port, storage)** - создаёт TCP-сокет, `bind`, `listen`, в бесконечном цикле `accept`. На каждое подключение запускает поток с `handle_client`.

**main()** - читает аргументы `--host`, `--port`, `--storage` и вызывает `serve`.

### client.py

**recv_line(conn)** - то же, что на сервере: приём одной строки протокола.

**recv_exact(conn, n)** - приём `n` байт файла после заголовка с размером.

**send_line(conn, text)** - отправка строки протокола с `\n`.

**cmd_list(conn)** - шлёт `LIST`, разбирает ответ (`OK`, число, список имён) и печатает имена в консоль.

**cmd_upload(conn, local_path)** - проверяет локальный файл, берёт его имя для сервера, шлёт `UPLOAD`, ждёт `READY`, отправляет размер и байты, печатает `OK` или ошибку.

**cmd_download(conn, remote_name, local_path)** - шлёт `DOWNLOAD`, при `OK <размер>` принимает байты и сохраняет в файл. Если второй аргумент не задан - сохраняет под именем с сервера.

**cmd_exit(conn)** - шлёт `EXIT`, печатает `BYE`, сигнализирует выход из цикла клиента.

**print_help()** - выводит список команд (только на стороне клиента, на сервер не отправляется).

**repl(host, port)** - создаёт сокет, подключается к серверу, в цикле читает ввод пользователя (`input`), вызывает `cmd_*` или `print_help`. Обрабатывает обрыв соединения.

**main()** - аргументы `--host`, `--port` и запуск `repl`.

## 2. Реализация

### Общая схема

Клиент и сервер общаются по TCP. Команды и статусы - текстовые строки. Содержимое файла - сырые байты, отдельно от строк, всегда после указания размера.

### Сервер

1. При старте создаётся сокет и папка `server_storage`.
2. Сервер слушает порт и ждёт подключений.
3. Каждый клиент обрабатывается в **отдельном потоке** (`threading.Thread`), поэтому несколько клиентов работают одновременно.
4. После отключения одного клиента главный цикл снова вызывает `accept` - сервер не останавливается.
5. Все файлы лежат только в `storage`; путь собирается как `storage + safe_basename`.

### Клиент

1. Подключается к IP и порту сервера.
2. Пользователь вводит команды в консоль.
3. Клиент переводит команду в строки протокола и читает ответы сервера.
4. При UPLOAD/DOWNLOAD клиент сам следит за размером и количеством принятых/отправленных байт.

### Протокол и файлы

- **LIST** - только строки, без бинарных данных.
- **UPLOAD** - после команды идёт фаза передачи: размер (строка), затем ровно столько байт. Пустой файл: размер `0`, байт нет.
- **DOWNLOAD** - одна строка `OK <размер>`, сразу после неё поток байт. Если файла нет - только `ERROR`, байт не следует.

### Обработка ошибок

- Неверная команда - ответ `ERROR`, сервер продолжает работу с этим клиентом.
- Обрыв во время чтения - сессия закрывается, процесс сервера жив.
- Нет локального файла (UPLOAD) - проверка на клиенте до отправки.
- Нет файла на сервере (DOWNLOAD) - `ERROR файл не найден`.
